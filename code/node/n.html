n - node 版本控制工具

Chris F 2020-11-26 node

# 全局安装

npm install -g n
1

# 安装最新稳定版 node 版本

n stable
1

# 使用 n 切换版本

n 
# 出现
  node/8.12.0
ο node/11.0.0
1
2
3
4

# n 切换 node 版本无效

可能原因 node的安装目录和 n 默认的路径不一样

# 查看 node 安装路径

which node
# /usr/local/bin/node
1
2

而 n 默认安装路径是 /usr/local，若你的 node 不是在此路径下，n 切换版本就不能把 bin、lib、include、share 复制该路径中，所以我们必须通过 N_PREFIX 变量来修改 n 的默认 node 安装路径。

# 使用vim 编辑 vi .bash_profile 文件，在结尾处添加两行

# 查看安装路径
which n

export N_PREFIX=/usr/local #根据你的安装路径而定
export PATH=$N_PREFIX/bin:$PATH
1
2
3
4
5

# 保存，刷新文件

source  .bash_profile
1

# 重新安装

n stable
1

# 重新切换版本即可，根据提示可能要授予 sudo 权限。

node -v 
# 11.0.0
1
